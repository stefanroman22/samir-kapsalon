# Corrections from the user

This file logs explicit corrections the user gave during builds. The agent reads it before every build phase. Append (never overwrite) when a new correction comes in.

## Format

Each entry uses this structure:

```
## YYYY-MM-DD — <one-line summary>

**Context:** what we were doing
**What I did:** what the agent did wrong
**Correction:** what the user said
**Applied:** what changed as a result
**Generalizable rule (if any):** the broader takeaway — if present, also append to conventions.md
```
